# CRM v2 Workflow Walk — 2026-04-21 Audit

Walked the app as Key would use it. Safety: observed only, no sends / archives / DNC.

## Key's daily workflow (priority order)

**Morning (7–9am):**
1. Open CRM → briefing modal pops → scan Urgent / Waiting / Stuck quotes / Overdue
2. Dismiss briefing → land on QUICK view (the new default)
3. Work the REPLY NOW section → open each waiting contact → reply
4. Work PERMITS IN FLIGHT → move each forward one step

**Mid-day (9–5):**
5. Inbound SMS toasts → open → reply
6. New leads arrive → Alex handles opener
7. Qualified → Q shortcut → QuickQuote modal → Create & Copy SMS
8. Install day → BRIEF button → drive out

**Evening (5–8):**
9. QUICK → STUCK QUOTES → draft follow-ups
10. FINANCE → outstanding / paid / overdue check
11. CALENDAR → confirm tomorrow's installs
12. SELECT mode → archive dead leads

**Weekly:**
13. PERMITS sub-view → filter by jurisdiction → batch portal submit
14. MATERIALS sub-view → order what's pending

## All Findings — Consolidated

### Already fixed in this session (shipped 2026-04-21)

- ✅ **Quick List** — new default sub-view with REPLY NOW / TODAY / THIS WEEK / PERMITS IN FLIGHT / STUCK QUOTES sections.
- ✅ **Unknown-name leakage** — `displayNameFor()` helper; fallback to `···8307` (phone last-4) across briefing, calendar, pipeline cards, permits table, quick list.
- ✅ **Street View fallback** — metadata endpoint check before rendering; "we have no imagery" gray tiles no longer leak through.

### Still to fix

#### High priority

**H1. Stage-vs-proposal divergence.**
Finance shows 20 proposals out, but LEADS chip says QUOTED=1 (contacts.stage=2). Proposal creation should bump `contacts.stage` to 2 if it's still 1. Otherwise the QUOTED chip is silently wrong.

**H2. Quick List ignores snooze.**
Found in walk: snoozed Charles Snyder for 1 day on the detail panel → toast confirmed, but he still showed at top of PERMITS IN FLIGHT on Quick List. Briefing honors `isSnoozedFor(id)`; Quick List needs the same filter.

**H3. Duplicate outbound Alex messages.**
Key's test contact has the same Alex opener sent 3× back-to-back. Retry/idempotency bug in the Alex edge function (not CRM v2 itself). Note for later.

#### Medium priority

**M1. Pipeline column headers truncate.**
"PERMIT SUBMIT…" / "READY TO P…" — can't identify columns. Abbreviate systematically.

**M2. Finance KPI cards not drill-through.**
"5 OVERDUE" / "$12,001 OUTSTANDING" — Key sees the number, can't click to see the list.

**M3. Materials ORDER column has no action.**
Status-only text. Should be an action button to mark parts ordered.

**M4. Quick Quote add-on buttons unclear.**
Cord / Surge / POM / Permit — POM is undefined. Cord default-on, others default-off. Inactive/active visual difference subtle.

**M5. Filter empty state is void.**
Tapping WAITING=0 chip shows blank list area — no "Nobody waiting on you. Nice work." hero. Confusing on first load.

**M6. Sparky markdown not rendered.**
Sparky's responses use `**bold**` syntax that renders as literal asterisks in the chat. Needs markdown-to-react rendering.

**M7. Sparky stage labels wrong.**
Sparky said "Stage 3 (Quote Sent)" but CRM stage 3 is BOOKED (QUOTED is stage 2). System prompt is mapping stages incorrectly.

**M8. Sparky date math off.**
Sparky said Apr 4 = "13 days ago" when today is Apr 21 (should be 17 days). Its "today" concept is stale.

**M9. Install Brief / Outstanding mismatch.**
Install Brief shows Deal $2,023 (Approved), but contact panel shows Outstanding $3,460. No breakdown reconciles the two. Key would wonder which is right.

**M10. Stuck-quote row opens Messages tab.**
Clicking a STUCK QUOTE row opens contact detail on Messages. Should land on Quote tab — Key's next action is to look at the quote and draft a follow-up.

#### Low priority

**L1. Briefing rows not clickable.**
"Tom — 18 days silent" is text-only. Tapping should open that contact.

**L2. Alex "handed off / resume" strip cryptic.**
Looks tappable but unclear which is state vs. action.

**L3. Permit # field shows internal ID.**
`pr: b6444745d5...` appears where permit number goes. Confusing — that's an internal UUID, not the city's permit number.

**L4. Name casing inconsistency.**
"Key" vs "key goodson" — depends on Alex transcript vs form fill. Normalize at write-time (title case).

**L5. No ⌘K hint in top bar.**
Command palette keyboard-only; new user wouldn't discover it. Small `⌘K` button (or "search" icon) in the top bar would help.

**L6. New Lead modal has no source tag.**
Adding a manual lead leaves no trace of where they came from (walk-up, referral, phone call, test).

**L7. Sparky mode tabs feel redundant.**
Chat / Briefing / Insight / Reply / Draft — all just swap suggestion chips. No mode-specific behavior otherwise.

**L8. Bulk-select indicator subtle.**
Selected row = small gold check top-left + navy left stripe. Easy to miss. Could be a full-row tint.

**L9. Dark mode contrast gaps.**
Row separators nearly invisible; stage chip text low-contrast on the darker row background.

### Good (keep)

- Stage filter chips with live counts on LIST view.
- Pipeline dollar-pipeline header ("69 active · $20,723 pipeline") — motivating.
- Permits table's 7-step checkbox grid + NEXT column — reads like a checklist.
- Calendar's AWAITING-DATE section makes slipped-through bookings obvious.
- Snooze 1d/3d/7d buttons — low-friction defer (toast + Clear).
- Install Brief modal — consolidates everything in one glance.
- BRIEF button on contact detail — one-click to brief from inside detail.
- Sparky chat is genuinely useful (real analysis, specific contacts ranked by urgency).
- Mobile contact detail fullscreen with `<` back — iOS-native feel.
